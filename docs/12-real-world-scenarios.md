# 실전 시나리오 (현실 문제 매핑)

**모드 이름 (UI)**: 실전 시나리오  
**문항 타입 (`type`)**: `scenario_map`  
**상세 스펙**: [03-content-types.md](03-content-types.md) § G

---

## 철학

- **목표는 회사 기출 암기가 아니라 매핑 훈련**이다. “배달 앱이 이렇게 물었던 문제”를 외우기보다, 일상·서비스 상황을 **제약·입출력·자료구조**로 바꾸는 연습을 한다.
- Pick·Blank가 **알고리즘 이름·코드 골격**을 익힌다면, 실전 시나리오는 **지문이 길고 도메인 말이 섞인** 상태에서 같은 패턴을 찾는다.
- 톤은 **가벼운 상황극**이다. PPT 케이스 스터디·면접 보고서 느낌은 피하고, 짧은 에피소드 + 선택지로 끝낸다 ([02-core-learning-loop](02-core-learning-loop.md) 재미 원칙).

## 카테고리 분류 (taxonomy)

문항 `metadata.category` 또는 `scenarioCategory`로 태깅한다. MVP는 아래 6~8개만 사용.

| category | 예시 상황 (일반화) | 자주 나오는 패턴 |
|----------|-------------------|------------------|
| `logistics` | 배송 경로, 픽업 순서, 거리 최소화 | BFS, 그리디, 다익스트라(Later) |
| `matching` | 추천, 페어링, 중복 제거 매칭 | 해시, 투 포인터, 정렬+투포인터 |
| `scheduling` | 회의실, 작업 큐, 시간표 | 정렬, 힙(Later), 시뮬레이션 |
| `search_filter` | 검색어, 필터, 페이지네이션 | 이분 탐색, 해시, 배열 순회 |
| `aggregation` | 일별 통계, Top-K, 롤링 합 | 해시, 누적합, 힙(Later) |
| `limits_security` | API 호출 제한, 중복 요청 차단 | 해시+시간, 슬라이딩 윈도우 |
| `inventory` | 재고, 좌석, 티켓 잔여 | 스택/큐, 해시 |
| `navigation` | 지도 타일, 연결 컴포넌트 | BFS, DFS |

실제 기업·플랫폼 이름·실제 서비스 UI·실제 기출 문구는 **사용하지 않는다**. “대형 메신저”, “배달 플랫폼” 수준의 일반 표현만 허용.

## 패턴 → 시나리오 치트시트 (제작·학습용)

| 패턴 태그 | 시나리오에서 드러나는 신호 | 학습자에게 묻는 질문 |
|-----------|---------------------------|---------------------|
| `array` | 목록 한 바퀴, 인덱스, 연속 구간 | “먼저 배열로 놓을 수 있나?” |
| `two_pointer` | 정렬된 목록, 양끝 좁히기, 구간 합 | “정렬 전제가 붙나?” |
| `hash` | 빠른 존재/빈도, 중복 제거, id 조회 | “키로 O(1) 조회가 필요한가?” |
| `binary_search` | 정렬된 값/시간축에서 경계 찾기 | “단조성·정렬이 보장되나?” |
| `stack` | 최근 것 취소, 괄호, 역순 처리 | “나중에 넣은 것을 먼저 꺼내나?” |
| `bfs` | 최소 단계, 격자/그래프 최단 | “칸 수·홉 수를 최소로?” |

World 1~2에서 다루는 6패턴만 MVP 정답 후보로 제한한다. DP·그리디 전용 시나리오는 Later.

## Daily · Algorithm · Level과의 연결

| 모드 | 연결 방식 |
|------|-----------|
| **Daily** | 주 1~2회 `scenario_map` 1문항 포함 가능 (Should). 나머지는 Pick+Blank |
| **Algorithm** | 각 알고리즘 상세 하단 “실전에 이렇게 나와요” 2~3문항 (v1.1) |
| **Level** | World 1 말미·World 2에서 스테이지 1~2개에 1문항씩 배치 ([07-curriculum-world-1](07-curriculum-world-1.md)) |
| **실전 시나리오 탭** | 카테고리 필터 + 난이도, 세션 3~5문항 (Should) |

학습 경로 권장: Pick으로 패턴 이름 익힘 → Blank로 골격 → **실전 시나리오로 지문 번역** → (Later) 외부 플랫폼 풀이.

## 콘텐츠 제작 가이드 (작가용)

### 지문

- 3~6문장. **역할·목표·제약**만 (데이터 규모, 정렬 여부, 최소/최대).
- 코드·수식 없이도 풀 수 있게. 필요 시 입력 예시 1줄.
- 유머·캐릭터 한 줄 OK (“라이더 민수”, “점심 메뉴 앱”) — 과한 스토리텔링은 금지.

### 선택지

- 패턴 선택: 4~6개, **정답 1~2개** (`primaryPatternIds`) + 그럴듯한 함정 2개 (비슷한 도메인, 다른 패턴).
- (선택) 모델링 1차: “먼저 무엇으로 모델링?” — `array` / `graph` / `hash_map` / `queue` 등 4지선다.

### 해설

- ① 상황을 알고리즘 언어로 한 줄 번역 ② 왜 이 패턴 ③ Pick/Blank 중 연습할 문항 힌트(태그만, ID는 선택).

### QA 체크리스트

- [ ] 60~90초 안에 읽고 고를 수 있는가
- [ ] 정답 패턴이 6종 MVP 태그 안에 있는가
- [ ] 특정 회사·실제 코테 문구를 복제하지 않았는가
- [ ] 해설에 “왜 다른 선택지는 애매한지” 1문장 포함
- [ ] 난이도 1: 신호가 명확 / 3: 두 패턴이 그럴듯하게 경쟁

### 난이도 곡선

| 단계 | 문항 수 (가정) | 특징 |
|------|----------------|------|
| v1.0 Should | 10~15 | 카테고리 4~5종, primary 패턴 1개만 채점 |
| v1.1 | +15 | `modelingQuestion` 활성, 복수 정답 패턴 |
| v2 | +20 | 2단계 시나리오, World 3+ |

## 예시 테마 (카카오 스타일 **유사**, 복제 아님)

| 테마 | 시나리오 한 줄 | 연결 패턴 |
|------|----------------|-----------|
| 배송 경로 | 여러 정류장을 한 번에 돌 때 이동 칸 수 최소 | BFS |
| 메뉴 추천 | 최근 본 메뉴와 겹치지 않게 다음 후보 | 해시 |
| 회의실 | 겹치지 않는 최대 회의 개수 (간단 버전) | 정렬+그리디 맛보기 → MVP는 `array`/`two_pointer`로 단순화 |
| API 제한 | 사용자당 1분에 100회 초과 요청 차단 | 해시 + 시간 윈도우 (`hash`) |
