# 콘텐츠 유형 스펙

MVP에서 다루는 5가지 핵심 모드, **실전 시나리오(G)**, 이후 브릿지(H)를 정의한다.

---

## A. Pick algorithm (알고리즘 고르기)

### 설명

짧은 지문(상황·제약)을 읽고, 맞는 **알고리즘/패턴**을 3~4개 선택지에서 고른다.

### UX

- 지문 상단, 선택지는 카드 또는 라디오 버튼
- 제출 → 즉시 정오 + 해설 접기/펼치기
- 난이도 표시: ★1~3 (선택)

### 학습 목표

- 문제를 **유형으로 분류**하는 속도 향상
- 유사 패턴 간 혼동 줄이기 (예: 이분 탐색 vs 투 포인터)

### MVP

- 약 50문항, 6패턴 태그
- World·Algorithm·Daily에서 재사용

---

## B. Fill-in-the-blank (코드 빈칸)

### 설명

5~15줄 Python 스켈레톤에 빈칸 1~3개. 드래그 블록 또는 키워드 픽(MVP).

### UX

- 코드는 고정폭, 빈칸은 강조 박스
- 블록 풀: 정답 + 오답 2~4개 (MVP는 키워드 픽으로 단순화 가능)
- 빈칸별 순차 제출 또는 한 번에 제출 (MVP: 한 번에 제출 권장)

### 학습 목표

- 패턴의 **전형적 코드 골격** 암기
- “이 줄이 없으면 왜 틀리는지” 연결

### MVP

- 약 30문항
- Blank당 최대 3칸, 한 파일 15줄 이내

---

## C. Level mode (월드·스테이지)

### 설명

게임 맵 형태의 **World → Stage** 진행. World 1은 배열·기초, World 2는 스택/큐·BFS 등.

### UX

- 맵에서 스테이지 노드 탭 → 미니 세션(문항 3~5)
- 잠금: 이전 스테이지 클리어 또는 별 N개
- 스테이지 클리어: 정답률 ≥ 70% 또는 하트 남김 (정책은 [05-mvp-scope](05-mvp-scope.md))

### 학습 목표

- 커리큘럼 순서대로 패턴 습득
- 성취감·탐험 동기

### MVP

- World 1~2, World 1은 [07-curriculum-world-1](07-curriculum-world-1.md) 참고

---

## D. Algorithm mode (알고리즘별 마스터리)

### 설명

BFS, DP, 이분 탐색 등 **알고리즘 단위**로 Pick+Blank 혼합 세트와 진행률 %.

### UX

- 알고리즘 카드 그리드 → 상세(진행 바, 문항 수)
- “연습 5문항” / “복습 (오답만)” (복습은 Should)

### 학습 목표

- 한 패턴에 대한 **집중 반복**
- 약한 패턴 식별 (오답 태그 기반)

### MVP

- 6알고리즘: array basics, two-pointer, hash, binary search, stack, BFS
- 진행 % = 해당 태그 문항 중 정답 처리 비율 (최소 1회 정답)

---

## E. Daily challenge (일일 챌린지)

### 설명

전 사용자 동일 시드 또는 날짜 기반 시드로 **고정 세트** (예: Pick 3 + Blank 2).

### UX

- 홈 최상단 카드 “오늘의 챌린지”
- 완료 시 스트릭 +1, 캘린더 도장
- 자정(로컬) 기준 리셋

### 학습 목표

- **매일 3~5분** 습관
- 전체 패턴 균형 복습

### MVP

- 1일 5문항 고정 구성
- 스트릭: 완료 기준 (문항 전부 시도 + 3개 이상 정답 등 — [04-gamification](04-gamification.md))

---

## G. Scenario map (실전 시나리오 · 현실 문제 매핑)

### 설명

**서비스·업무 맥락**이 담긴 짧은 시나리오를 읽고, 적용할 **알고리즘/패턴**을 고른다. (선택) “먼저 무엇으로 모델링할까?” 1문항. 코딩 테스트의 “현실 문제”처럼 지문이 길지만, 답은 여전히 **패턴 선택**이다.

상세 철학·카테고리·작가 가이드: [12-real-world-scenarios.md](12-real-world-scenarios.md).

### UX

1. **시나리오 카드** — 3~6문장, 일러스트/아이콘(선택), 카테고리 뱃지 (예: 배송, 추천)
2. **패턴 고르기** — 4~6개 카드, **단일 또는 복수 선택** (MVP: primary 1개만 채점; Should: 복수+순위)
3. (선택) **모델링 1차** — “먼저 어떤 자료구조/관점으로 볼까?” 4지선다 (`array` / `graph` / `hash_map` / `queue` 등)
4. 제출 → 정오 + 해설 (상황을 알고리즘 문장으로 번역 + Pick/Blank 태그 힌트)

톤: 드라이한 케이스 스터디가 아니라 **짧은 상황극** ([02-core-learning-loop](02-core-learning-loop.md) 재미 원칙). 캐릭터 한 줄·가벼운 유머 허용.

### 학습 목표

- 현실 지문에서 **제약·입출력·자료구조** 신호 읽기
- Pick에서 익힌 패턴 이름을 **도메인 언어**에 연결
- “구현 전에 무엇으로 모델링할지” 1차 습관 (선택 문항)

### 난이도·진행

| difficulty | 지문·선택지 |
|------------|-------------|
| 1 | 신호가 명확 (정렬됨, 최소 칸 수 등) |
| 2 | 두 패턴이 그럴듯하게 경쟁 |
| 3 | 제약 2개 이상, 모델링 보조 문항 포함 |

- **World 1**: 스테이지 1~2곳에 맛보기 1문항 ([07-curriculum-world-1](07-curriculum-world-1.md))
- **World 2·실전 탭**: 비중 확대 (BFS·스택 시나리오)

### Pick·Blank와의 관계

| | Pick / Blank | 실전 시나리오 |
|--|--------------|----------------|
| 지문 | 알고리즘 용어 위주 | 서비스·업무 맥락 |
| 응답 | 1패턴 또는 코드 빈칸 | 패턴(+선택적 모델링) |
| 역할 | 이름·골격 암기 | **번역·매핑** |

같은 `tags`로 Algorithm·Daily와 묶는다. 시나리오 해설 끝에 “이 패턴 Pick 복습” 링크(Should).

### 예시 테마 (일반화, 특정 회사·기출 비복제)

| 테마 | 상황 스케치 | 연결 패턴 |
|------|-------------|-----------|
| 배송·경로 | 정류장/격자에서 이동 **칸 수 최소** | BFS |
| 추천·매칭 | 본 적 있는 항목 제외, 빠른 조회 | 해시 |
| 스케줄 | 겹치는 구간, 시간순 처리 | 정렬·배열 (MVP 단순화) |
| 호출 제한 | 사용자별 분당 N회 초과 차단 | 해시 + 시간 창 |

### MVP

- **Should (v1.0~1.1)**: 문항 10~15, 전용 탭 또는 Learn Hub 진입 ([05-mvp-scope](05-mvp-scope.md))
- Must 출시 차단에는 넣지 않음 — Pick/Blank 안정 후 투입

## H. Bridge to external (Later)

### 설명

학습한 패턴과 연결된 **백준/프로그래머스 문제 ID** 추천. 앱 안에서는 링크만.

### UX

- Algorithm 모드 하단 “실전 문제 풀기”
- 난이도·패턴 태그 필터

### MVP

- **제외** (문서만 placeholder)

## 모드 간 문항 재사용

| 문항 ID | Level | Algorithm | Daily | Scenario |
|---------|-------|-----------|-------|----------|
| 동일 JSON | ○ 스테이지에 배치 | ○ 태그로 필터 | ○ 날짜 시드로 샘플링 | ○ 카테고리·태그 필터 |

`scenario_map`은 Pick/Blank와 **별도 JSON**이지만 `tags`, `difficulty`는 동일 규칙 ([06-content-schema](06-content-schema.md)).

콘텐츠 제작 시 `tags`, `difficulty`, `type` 필수 ([06-content-schema](06-content-schema.md)).

## 문항 품질 체크리스트

- [ ] 지문만 읽고 60초 안에 답을 고를 수 있는가
- [ ] 정답이 1개로 명확한가
- [ ] 해설이 패턴 이름을 포함하는가
- [ ] Blank 오답 선택지가 그럴듯한가 (함정)
- [ ] Python 3.10+ 문법, 실행 가능한 스켈레톤인가
- [ ] Scenario: 90초 안에 읽을 수 있고, 회사·실제 기출 문구를 복제하지 않았는가
- [ ] Scenario: 해설이 “상황 → 패턴” 번역을 포함하는가
